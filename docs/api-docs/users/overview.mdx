# GraphQL Users API Overview

## Introduction

The GraphQL Users API is an account-level management GraphQL API that allows you to create, edit, and delete BigCommerce control panel users. This highly requested feature is for Enterprise merchants who need to audit existing users that can also integrate with third-party user management systems.

## What is a user

A user is an account that can sign in to the BigCommerce control panel for an account or store.

## Prerequisites

Before getting started with the GraphQL Users API, make sure you have the following:

* BigCommerce store
* A valid API account access token with sufficient permissions to make the desired requests

## OAuth scopes

| Name | Permission | Parameter | Description |
|:-----|:-----------|:----------|:------------|
| Account details  | read-only | `account-read` | View account details |
| Account users | read-only | `account_users_read`| View the users associated with an account |
| Account stores | read-only | `account_stores_read` | View the stores associated with an account |
| Account apps| read-only | `account_apps_read` | View the apps associated with an account |
| Add user to account | modify | `account_users_write` | View and modify adding a user from an account |
| Remove user from account | modify | `account_users_write` | View and modify removing a user from an account |
| Create user | modify | `account_users_write` | View and modify a user |
| Add user to store | modify | `account_users_write` | View and modify adding a user to a store |
| Remove user from store | modify | `account_users_delete` | View and modify removing a user from a store |

For more information on available authentication methods, see [Authentication](https://developer.bigcommerce.com/api-docs/getting-started/authentication).

## Getting started

The GraphQL Users API introduces a new way to manage users from within your BigCommerce account. Using an OAuth-based API account, you can integrate our APIs with your current system to create a custom experience.

Begin with the following steps to get started:

1. Request an access token and account UUID from Settings -> Account API tokens. Note this is a different token than Store API token.
2. Use a GraphQL client such as [Altair](https://altairgraphql.dev/) to test the API.
3. In the client, set the request URL to `https://api.bigcommerce.com/accounts/{account_uuid}/graphql`.
4. Send an `X-Auth-Token` request header with the access token as its value.
5. Use the request body to send the GraphQL query or mutation you want to perform.

## Example requests

This section contains examples of GraphQL Users API queries and mutations that you can send with the following HTTP configuration:

```http filename="GraphQL Users API HTTP configuration" showLineNumbers copy
POST https://api.bigcommerce.com/accounts/{{account_uuid}}/graphql
X-Auth-Token: {{access_token}}
Accept: application/json
Content-Type: application/json
```
